# E-Commerce Sales Analysis using Python & SQL

An end-to-end data analytics project that answers real-world e-commerce business questions by querying a MySQL database directly from Python and visualizing the results with Matplotlib and Seaborn.

## 📌 Project Overview

This project analyzes an e-commerce dataset (orders, customers, products, order items, and payments) stored in a MySQL database. Using Python as the interface, I wrote and executed SQL queries — ranging from basic aggregations to advanced window functions — to extract business insights around sales, customer behavior, seller performance, and revenue trends.

The goal was to practice and demonstrate strong SQL querying skills (joins, subqueries, CTEs, window functions) combined with Python-based data handling and visualization.

## 🛠️ Tech Stack

- **Python** – scripting and orchestration
- **MySQL** – database and query engine
- **Pandas** – transforming SQL query results into DataFrames
- **Matplotlib & Seaborn** – data visualization
- **NumPy** – correlation analysis

## 🗄️ Dataset

A relational e-commerce dataset with the following core tables:
- `customers` – customer_id, customer_city, customer_state
- `orders` – order_id, customer_id, order_purchase_timestamp
- `order_items` – order_id, product_id, seller_id, price
- `products` – product_id, product_category
- `payments` – order_id, payment_value, payment_installments

## ⚙️ Steps I Followed

### 1. Environment Setup & Database Connection
- Set up a local MySQL database (`ecommerce`) and loaded the raw tables (customers, orders, order_items, products, payments).
- Connected to MySQL from Python using `mysql.connector`, and created a cursor to execute SQL queries directly from the notebook.
- Imported `pandas`, `matplotlib.pyplot`, `seaborn`, and `numpy` for downstream processing and visualization.

### 2. Basic Queries — Foundational Exploration
- Listed all unique cities where customers are located.
- Counted the total number of orders placed in 2017.
- Calculated total sales per product category to identify max and min performing categories.
- Calculated the percentage of orders paid in installments.
- Counted the number of customers per state and visualized it with a bar chart.

### 3. Intermediate Queries — Trend & Behavior Analysis
- Calculated the number of orders placed per month in 2018 and visualized seasonal ordering trends.
- Found the average number of products per order, grouped by customer city, using a CTE.
- Calculated each product category's percentage contribution to total revenue.
- Investigated the correlation between average product price and purchase frequency using `numpy.corrcoef` (found a near-zero correlation, indicating no strong relationship between the two).
- Calculated total revenue generated by each seller and ranked them using `DENSE_RANK()`.

### 4. Advanced Queries — Window Functions & Business Metrics
- Calculated a 3-order moving average of payment value for each customer using `AVG() OVER (PARTITION BY ... ROWS BETWEEN 2 PRECEDING AND CURRENT ROW)`.
- Calculated cumulative monthly sales per year using a running `SUM() OVER (ORDER BY ...)`.
- Calculated year-over-year sales growth rate using `LAG()` window function.
- Measured customer retention rate — the percentage of customers who placed a second order within 6 months of their first order — using CTEs and a self-join pattern.
- Identified the top 5 highest-spending customers for each year using `DENSE_RANK()` partitioned by year, and visualized the results as a grouped bar chart.

### 5. Visualization
- Used Matplotlib for quick bar charts (e.g., customers by state).
- Used Seaborn for more polished, labeled visualizations (e.g., orders per month with bar labels, top customers by year with hue-based grouping).

## 📊 Key Insights and Improvement

- Order volume and revenue show clear monthly/seasonal variation (Nearly no purchasing history from September onwards).
- Product price and purchase frequency are essentially uncorrelated — pricing alone doesn't drive purchase volume.
- Revenue is concentrated among a subset of top sellers and top customers.
- Customer retention within a 6-month window was found to be "None" in this dataset, highlighting a potential area for a retention-focused business strategy.
- 99% of the payments were installment-based, indicating strengthening partnerships with installment payment providers may encourage higher-value purchases and improve customer retention.
